# Aufgaben

## 1. Zusammenfassung der Theorie

Zuerst haben wir einen neuen Order `260324` mit dem command `mkdir 260324` erstellt und sind in diesem Verzeichnis in Visual Studio Code gegangen.

In Visual Studio Code haben wir dann eine Shell Skript Datei erstellt.

In der ersten Zeile steht dann das 
`#!/bin/env bash`, was festlegt dass Bash verwendet wird.

Mit `echo` gibt man dann die jeweiligen Texte und Variablen aus. Wobei man bei Variablen aber im echo ein `$` davorstellen muss.

Die **Leerzeichen** geben den Schluss eines command Teil an also sollte man darauf aufpassen sie nicht zu verwenden wo sie nicht hingehören.

Dateien kann man mit `>` erstellen oder überschreiben wie hier: `echo "Zeynep yay > $DIR_NAME/$FILE_NAME`

Wenn man Variablen verwendet muss man double scores `"` verweden denn wenn man nur single scores `'` verwendet funktionieren die Variablen nicht.

Wenn man Variablen direkt mit Text verwendet muss curly braces `{}` um den filename machen. Also zum Beispiel: `"${FILE}_002.txt"`

Um diese Dateien dann auszugeben verwendet man: `./hello.sh`, was in vielen Fällen aber nicht funktioniert, da die Permissions nicht mit `execute` also dem x ausgestattet sind und man sie somit auch nicht ausführen kann.

Also muss man den command `chmod +x hello.sh` verwenden um die Permissions zu ändern und für jede Gruppe das `x` für `execute` hinzufügen.

Kommentare macht man mit einem `#` davor.

## 2. Übung (Variablen)

**Löse diese Aufgabe ohne ein shell-script zu schreiben.**

**Lege 2 Variablen für Directories an, eine Variable für den Filenamen und eine Variable für die Dateiendung (Extension).**

**Verwende nur die Variablen anstatt der Directory und Filenamen um folgendes zu lösen:**

- **Erzeuge die Directories (das zweite befindet sich in dem ersten).**
- **Lege in dem zweiten Directory eine Text-Datei mit Inhalt an, verwende Filenamen und Extension (getrennt mit .).**
- **Gib den Inhalt der Datei (unter Verwendung der Variablen) aus.**

Zuerst mal die Variablen anlegen im Terminal in Kali:

```
DIR_1=ordner_1
DIR_2=ordner_2
FILE=myfile
EXIT=txt
```

Dann die Verzeichnisse erstellen:
`mkdir -p $DIR_1/$DIR_2`

Daraufhin die Datei erstellen und einen Inhalt reinschreiben:
`echo "Was geht" > $DIR_1/$DIR_2/${FILE}.${EXIT}`

Danach den Inhalt der Datei ausgeben:
`cat $DIR_1/$DIR_2/${FILE}.${EXIT}`

und der Inhalt ist ja `Was geht`, was ich vorher reingeschrieben habe.

## 3. Übung (Begrüßung)

**Schreibe ein Shell-Script mit dem Namen begruessung.sh, dieses soll:**

1. **den Benutzer nach seinem Namen fragen**
1. **den Namen einlesen und in einer Variable speichern**
1. **danach ausgeben:**

```
Hallo <NAME>
Schön, dass du da bist.
```
**Starte das Script auf alle besprochene Arten.**

**Hinweis: Verwende den `read` Befehl – recherchiere wie dieser zu verwenden ist.**

Zuerst mal die neue Shell mit dem namen `begruessung.sh` anlegen per VS Code.

Dann schreibt man den Code, welcher lautet:
```
#!/bin/env bash

echo "Wie lautet dein Name?"
read NAME

echo "Hallo $NAME"
echo "Schön, dass du da bist."
```

Die erste Zeile wieder um `bash` auszuwählen und dann den `echo` als Frage und dann die Variable `NAME` einlesen. Danach wieder mit `echo` die Variable auslesen.

Danach schaut man mit `ls -l` welche permissions `begruessung.sh` hat:

```
┌──(kali㉿kali)-[~/260324]
└─$ ls -l            
total 12
-rw-rw-r-- 1 kali kali  132 Mar 24 03:59 begruessung.sh
-rwxrwxr-x 1 kali kali  513 Mar 24 03:44 hello.sh
drwxrwxr-x 2 kali kali 4096 Mar 24 03:17 mydir
```

Da kann man sehen, dass die Datei nichts ausführen kann, weil es kein `execute` hat. Um das hinzuzufügen, gibt man den command:
`chmod +x begruessung.sh` ein. 

Wenn man jetzt wieder `ls -l` eingiebt, sieht man: 
```
──(kali㉿kali)-[~/260324]
└─$ ls -l
total 12
-rwxrwxr-x 1 kali kali  107 Mar 24 03:59 begruessung.sh
-rwxrwxr-x 1 kali kali  513 Mar 24 03:44 hello.sh
drwxrwxr-x 2 kali kali 4096 Mar 24 03:17 mydir
```

wenn man dann `./begruessung.sh` eintippt, sieht man:
```
┌──(kali㉿kali)-[~/260324]
└─$ ./begruessung.sh
Wie lautet dein Name?
Leah
Hallo Leah
Schön, dass du da bist.
```

Also der `read` Befehl liest die Eingabe vom Benutzer und speichert sie in die Variabel `NAME`.

## 4. Übung (Zeilenumbruch)

**Erzeuge eine Variable MSG, die mit `echo $MSG` folgende Ausgabe erzeugt:**
```
Name: <Dein Name>
Klasse: 3AHITS
Raum: ...
```
**Vorgaben/Hinweise:**

- **Es darf nur eine Variable verwendet werden.**
- **Der Zeilenumbruch muss in der Variablen mit `\n` definiert sein.**

Erstmal wieder die erste Zeile:
`#!/bin/env bash`

Danach die Variable `MSG` definieren:
`MSG="Name: Leah\nKlasse: 3AHITS\nRaum: 233"`

Das `\n` steht für den Zeilenumbruch aber nur wenn man bei der Ausgabe `-e` verwendet. Würde man kein `-e` schreiben würde das `\n` im Text stehen bleiben.

Also danach: `echo -e $MSG`

Daraufhin im Terminal wieder die Permissions mit `chmod` ändern:
`chmod +x hih.sh  `

und ausgeben, was dann so aussieht:
```
┌──(kali㉿kali)-[~/260324]
└─$ ./hih.sh       
Name: Leah
Klasse: 3AHITS
Raum: 233
```

## 5. Übung (admin)

**Gegeben ist:**
```
USER="admin"
```
1. **Gib folgende Strings mit echo korrekt aus:**
- **admin_backup**
- **admin_2026**

2. **Erkläre, warum folgende Zeile nicht das gewünschte Ergebnis liefert:**
```
echo "$USER_backup"
```

Diese Strings gibt man wie folgt aus:
```
#!/bin/env bash

USER="admin"

echo "${USER}_backup"
echo "${USER}_2026"
```

Also man gibt dem String `admin` erstmal eine Variable `USER` und gibt diese dann mit `echo` aus. Die curling strings benötigt man, weil das Terminal sonst nicht versteht, dass mit dem `USER` die Variable gemeint ist sondern es zu dem String eghört.

Also muss man den variablennamen mit curling strings umklammern. Damit es auch als Variable gekennzeichnet wird!

So bekommt man jetzt das hier raus:
```┌──(kali㉿kali)-[~/260324]
└─$ ./hih.sh
admin_backup
admin_2026
```
