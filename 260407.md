## 1. Übung (Direktory Struktur)

**Schreibe ein Shell-Script das Unterverzeichnisse und Dateien anlegt, es wird ein Argument an das Script übergeben.**

**Aufruf: `./build_dirs.sh abcd`**

**Dies führt zu folgender Directorystruktur:**
```
./
└── abcd/
    ├── abcd_01/
    │   ├── abcd.01.1.txt
    │   ├── abcd.01.2.txt
    │   └── abcd.01.3.txt
    └── abcd_02/
        ├── abcd.02.1.txt
        ├── abcd.02.2.txt
        └── abcd.02.3.txt
```

**Schreibe weiters ein Shell-Script `clean_dir.sh` das diese Verzeichnisstruktur wieder löscht.**

**Hinweis: Das Kommando `tree` kann verwendet werden um eine Directory Struktur in obiger Form darzustellen.**

---
Also erstmal wieder eine `.sh` Datei erstellen in VS Code:
```
#!/bin/bash

name=$1

if [ -z "$name" ]; then
  exit 1
fi

mkdir "$name"

for i in 01 02
do
  mkdir "$name/${name}_$i"

  for j in 1 2 3
  do
    touch "$name/${name}_$i/${name}.$i.$j.txt"
  done
done
```
Hier wird erstmal geschaut, ob denn schon etwas drinnensteht und dann wird ein neues Verzeichnis erstellt und in diesem dann auch Unterverzeichnisse.

Danach diese Script im Terminal ausführbar machen mit dem Command `chmod +x build_dirs.sh`

Daraufhin führt man den command aus mit dem command `./build_dirs.sh abcd` und danach `tree` eingeben. Dann kommt diese Struktur raus:
```

└─$ tree             
.
├── 260324
├── 260324.md
├── 260407.md
├── abcd
│   ├── abcd_01
│   │   ├── abcd.01.1.txt
│   │   ├── abcd.01.2.txt
│   │   └── abcd.01.3.txt
│   └── abcd_02
│       ├── abcd.02.1.txt
│       ├── abcd.02.2.txt
│       └── abcd.02.3.txt
├── build_dirs.sh
```

Jetzt eine Script-Datei `clean_dir.sh`, dass das Verzeichnis wieder löscht, erstellen:
```
#!/bin/bash
 
name=$1
 
if [ -z "$name" ]; then
  exit 1
fi

rm -r "$name"
```

Und dann hier auch wieder ausführbar machen, also `execute` anmachen mit `chmod +x clean_dir.sh` und danach mit `./clean_dir.sh abcd` ausführen und das Verzeichnis wurde wieder gelöscht!

## 2. Übung (Skript Generator)

**Schreibe ein Skript das**

- **eine Skriptdatei erzeugt (Name wird als Argument übergeben),**
- **die She-Bang Zeile einfügt,**
- **einen echo Befehl einfügt und**
- **das eXecution Flag für das Skript setzt.**

**Anwendung:**
```
$ ./makescript.sh mytest
```

**erzeugt die Datei `mytest.sh`:**
```
#!/bin/env sh

echo "mytest Skript"

# write yor script here
```

**Das erzeugte Skript kann sofort ausgeführt werden:**
```
$ ./mytest.sh
```
---

Ertsmal die Script-Datei `makescript.sh` erstellen, wo der Inhalt Shebang benutzt, echo einfügt und Name als Argument verwendet:
```
#!/bin/bash

if [ -z "$1" ]; then
  echo "Es wird kein Name angegeben"
  exit 1
fi

filename="$1.sh"

echo "#!/bin/env sh" > "$filename"
echo "" >> "$filename"
echo "echo \"$1 Skript\"" >> "$filename"
echo "" >> "$filename"
echo "# write your script here" >> "$filename"

chmod +x "$filename"

echo "Skript $filename wurde erstellt!"
```

In diesem code wird zuerst geschaut ob ein Name überhaupt eingegeben wurde und dann wird eine neue Datei erstellt, in diese wird dann ein Inhalt geschrieben und danach wird die Datei ausführbar gemacht und danach wird gesagt das jenes Skript erfolgreich erstellt wurde.

Jetzt geht man wieder ins Terminal und macht es ausführbar mit `chmod +x makescript.sh` und führt es aus mit `./makescript.sh mytest`.

Hier kommt dann erstmal die Ausgabe:
```                                                                    
┌──(kali㉿kali)-[~/SYTB/3AHITS-SYTB-Petereder-Leah]
└─$ ./makescript.sh       
Es wird kein Name angegeben
```

weil noch kein `mytest.sh` erstellt wurde. Also erstellen wir die Skriptdatei `mytest.sh`:
```
#!/bin/env sh
 
echo "mytest Skript"
```

Danach führt man beide gemeinsam aus, was uns dann eine Ausgabe gibt dass es erstellt wurde!
```
┌──(kali㉿kali)-[~/SYTB/3AHITS-SYTB-Petereder-Leah]
└─$ ./makescript.sh mytest 
Skript mytest.sh wurde erstellt!
```

## 3. Übung (Headline Cat)

**Verwende `$@` zur Lösung dieser Aufgabenstellung.**

**Schreibe ein Skript das eine Art `cat` zur Verfügung stellt. Als Argumente werden eine beliebige Anzahl von Textdateien übergeben. Das Ergebnis (der Inhalt aller dieser Dateien) wird in die Datei `result.txt` (fixer Dateiname) geschrieben (ist die Datei vorhanden soll deren Inhalt überschrieben werden). Jedem Datei-Inhalt soll eine Überschrift vorangestellt werden.**

**Beispiel – der Aufruf**
```
$ ./headline_cat.sh file1.txt file2.txt file3.txt
```
**ergibt die Datei result.txt mit folgendem Inhalt:**
```
== file1.txt ==========================================
Das ist der Inhalt
der ersten Textdatei

== file2.txt ==========================================
Das ist der Inhalt
der zweiten Textdatei

== file3.txt ==========================================
Das ist der Inhalt
der dritten Textdatei
```
---

Weider eine Script-Datei `headline_cat.sh` erstellen:
```
#!/bin/bash

if [ $# -eq 0 ]; then
  echo "Bitte mindestens eine Datei angeben!"
  exit 1
fi

> result.txt

for file in "$@"
do
  echo "== $file ==========================================" >> result.txt
  cat "$file" >> result.txt
  echo "" >> result.txt
done

echo "Fertig! Ergebnis in result.txt"
```

Hier wird geprüft, ob mindestens eine Datei überhaupt übergeben wurde. danach wird `result.txt` erstellt.

Danach wieder ausführbar machen und dann auch ausfhühren:
```
chmod +x headline_cat.sh
./headline_cat.sh file1.txt file2.txt file3.txt
```

Dann  wenn man `cat` eingibt,kommt dieser Inhalt:
```
┌──(kali㉿kali)-[~/SYTB/3AHITS-SYTB-Petereder-Leah]
└─$ cat result.txt                    
== file1.txt ==========================================

== file2.txt ==========================================

== file3.txt ==========================================
```

## 4. Übung (RANDOM)

**Schreibe ein shellscript das eine beliebige Menge von Dateinamen als Parameter akzeptiert. Von jeder dieser Dateien soll eine Kopie im gleichen Verzeichnis angelegt werden. Die Kopie unterscheidet sich vom Original durch eine angefügte Zufallszahl, Beispiel:**
```
test.txt --> test.txt.38573
```

**Aufrufbeispiele:**
```
$ ./randcp.sh test1.txt test2.txt
$ ./randcp.sh xyz1.md test3.txt abcd.dat
$ ./randcp.sh *.md
```

- **Hinweis: `$RANDOM` liefert bei jeder Verwendung eine zufällige Zahl.**
- **Achtung: `#!/bin/bash` in der she-bang Zeile verwenden. `sh` unterstützt (in REPL) keine Zufallszahlen.**

---

Die Script-Datei `randcp.sh` erstellen:
```
#!/bin/bash

if [ $# -eq 0 ]; then
  echo "Bitte Dateien angeben!"
  exit 1
fi

for file in "$@"
do
  if [ -f "$file" ]; then
    rand=$RANDOM
    cp "$file" "$file.$rand"
  else
    echo "Datei $file existiert nicht!"
  fi
done
```

Dann wieder ausführbar machen und dann auch ausführen:
```
chmod +x randcp.sh
./randcp.sh test.txt
```

Man muss zuerst natürlich etwas in den `test.txt` reinschreiben und dann wenn man es ausführt `ls` machen und dann sieht man zusätzlich eine Datei mit demselben Namen nur mit random Zahlen dabei:
```
test.txt
test.txt.23564
```

wenn man diese dann mit `cat` ausführt, steht derselbe Inhalt, wie im `test.txt`:
```
┌──(kali㉿kali)-[~/SYTB/3AHITS-SYTB-Petereder-Leah]
└─$ cat test.txt
Hallo


┌──(kali㉿kali)-[~/SYTB/3AHITS-SYTB-Petereder-Leah]
└─$ cat test.txt.23564
Hallo
```
