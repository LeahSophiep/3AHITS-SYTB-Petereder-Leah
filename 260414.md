# Theorie

Berechnungen (`+, -, *, /, %`) können mit der Syntax `$(( EXPR ))` durchgeführt werden:
```
# Berechnungen
X=99999
Y=88888
Z=$(( X - Y ))
echo $Z
```
Das Ergebnis lautet dann: `11111`

## 1. Übung (Anzahl Einträge in einem Verzeichnis)

**Schreibe ein shellscript das als Argument einen Pfad auf ein Verzeichnis erhält. Das Script soll die Anzahl der Einträge in diesem Verzeichnis als Zahl ausgeben. Dies soll aus der ls Ausgabe mit wc ermittelt werden.**

```
$ ./nbrentries.sh ~/test
Es sind 12 Einträge im dir ~/test
```

**Hinweis – Vergleiche die Ausgaben von**

```
ls
ls | cat
echo "$(ls)"
```
**Dokumentiere den Unterschied, was ist die Logik dahinter.**

---

Zuerst wieder Kali hochfahren.

Zuerst erstellt man wieder eine `.sh` Datei, in diesem Fall nennt man sie `nbrenties.sh` und schreibt den Befehl da rein, welcher lautet:

```
#!/bin/bash
 
DIR=$1

COUNT=$(ls "$DIR" | wc -l)
 
echo "Es sind $COUNT Einträge im dir $DIR"
```
Zuersrt speichert man hier das Ergument dann zählt man die Anzahl der Einträge und daraufhin gibt man es aus.

Danach geht man in cmd um das Script mit `chmod +x nbrenties.sh` ausführbar zu machen.

Danach führt man das schell script mit `.nbrenties.sh ~/test` aus und bekommt danach diese Ausgabe:
```
┌──(kali㉿kali)-[~]
└─$ ./nbrenties.sh ~/test   
Es sind 0 Einträge im dir /home/kali/test
```

Jetzt zu den **Unterschieden in den Varianten**.
Bei `ls` ist die Ausgabe in Spalten formatiert. Bei `ls | cat` wird jede Datei in eine eigene Zahl geschrieben und bei `echo "$(ls)"` wird alles zu einem String.

Die Logik dahinter ist, dass `ls` für Menschen formatiert, während `ls | cat` rohe Daten in jeder einzelnen Zeile hat und in `echo "$(ls)"` die Ergebnisse als Text in Variablen gespeichert werden.

## 2. Übung (Tage bis zum Ball)

**Schreibe ein Shell-Skript das die Anzahl der Tage bis zum HTL Ball ermittelt. Die Ausgabe soll in der folgenden Form sein.**

```
Es sind noch 42 Tage bis zum HTL Ball (2027-01-16)
```

**Hinweise:**
- **Verwende das date Format für seconds since the Epoch (1970-01-01 00:00 UTC)**
---

Wieder ein Shell-Script schreiben mit dem Inhalt:
```
#!/bin/bash

TARGET="2027-01-16"

NOW=$(date +%s)

TARGET_SEC=$(date -d "$TARGET" +%s)

DIFF_SEC=$(( TARGET_SEC - NOW ))

DAYS=$(( DIFF_SEC / 86400 ))

echo "Es sind noch $DAYS Tage bis zum HTL Ball ($TARGET)"
```
Hier holt man sich zuerst die aktuelle Zeit dann wandelt man diese Zeit in sekunden um und berechnet dann die Differenz, also wie viele Sekunden dazwischen liegen. Daraufhin rechnet man die Sekunden in Tage um.

Das habe ich dann `tagebisball.sh` genannt.

Dann wieder ausführbar machen `chmod +x tagebisball.sh` und dann auch ausführen:
```
┌──(kali㉿kali)-[~]
└─$ ./tagebisball.sh
Es sind noch 276 Tage bis zum HTL Ball (2027-01-16)
```

## 3. Übung (Zufälliger Satz)

**Erstelle ein Skript das einen Satz aus 5 zufälligen Wörtern bildet, z.B.:**

```
richtig ganzen Kilometer auf Mittel
```

**Wähle die Wörter aus dieser Wortliste.**

- **Hinweis: Das Komando sed -n 5p datei.txt gibt die 5te Zeile aus.**
- **Erweiterung: Das Skript soll auch die Anzahl der in der Wortliste enthaltenen Wörter ermitteln. D.h. die Konstante 1000 soll im Skript nicht vorkommen.**
---

Zuerst erstellt man eine `nano woerter.txt` Datei mit Woertern drinnen:
```
┌──(kali㉿kali)-[~]
└─$ cat woerter.txt
Zeynep
Leah
HTL
Cybersecurity
SYTB
laufen
sehen
kaufen
```
Dann die `.sh` Datei erstellen, die ich jetzt `zufallssatz.sh` genannt habe, mit dem Inhalt:
```
#!/bin/bash
 
FILE="woerter.txt"

COUNT=$(wc -l < "$FILE")

W1=$(sed -n "$(( RANDOM % COUNT + 1 ))p" "$FILE")
W2=$(sed -n "$(( RANDOM % COUNT + 1 ))p" "$FILE")
W3=$(sed -n "$(( RANDOM % COUNT + 1 ))p" "$FILE")
W4=$(sed -n "$(( RANDOM % COUNT + 1 ))p" "$FILE")
W5=$(sed -n "$(( RANDOM % COUNT + 1 ))p" "$FILE")

echo "$W1 $W2 $W3 $W4 $W5"
```

Also zuerst zählt man die Anzahl der Wörter in diesem File, danach generiert man eine Zufallszahl zwischen 1 und der Anzahl von Wörtern, danach holt man sich das Wort und gibt die 5. Zeile aus.

Zeurst macht man es dann wieder ausführbar `chmod +x zufallssatz.sh` und führt es daraufhin wieder mit `./zufallssatz.sh` aus. Da erscheint dann folgende Ausgabe:
```
┌──(kali㉿kali)-[~]
└─$ ./zufallssatz.sh
Leah kaufen Cybersecurity kaufen SYTB
```

## 4. Übung(dated copy V1)

**Create a script which will take a filename as its first argument and create a dated copy of the file. eg. if our file was named `file1.txt` and today is the 2021-10-29 it would create a copy such as `2021-10-29_file1.txt`. (To achieve this you will probably want to play with command substitution and the command `date`)**

---

Wieder eine `.sh` Skript-Datei erstellen. Ich nenne sie jetzt `datedcopy.sh` mit dem Inhalt:
```
#!/bin/bash

FILE=$1; 
DATE=$(date +%Y-%m-%d); 
cp "$FILE" "${DATE}_$FILE"; 
echo "Fertig: ${DATE}_$FILE"
```

Also zuerst holt man sich das aktuelle Datum, danach giebt man der Datei ein Argument und dann kopiert man es.

Dann erstellt man die Datei `file1.txt` :
```
echo "Hallo" > file1.txt
```

Danach wieder ausführbar machen mit `chmod +x datedcopy.sh` und dann ausführen `./datedcopy.sh file1.txt` dann erhält man diesen Inhalt:
```
┌──(kali㉿kali)-[~]
└─$ ./datedcopy.sh file1.txt
Fertig: 2026-04-14_file1.txt
```
