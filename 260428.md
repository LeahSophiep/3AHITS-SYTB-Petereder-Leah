# Zusammenfassung der Theorie

Jedes Programm liefert beim beenden einen `Exit Status`. Der Exit Status wird in die Variable `$?` gespeichert. Eine `0` bedeutet das die Ausführung fehlerfrei war und `ungleich 0` ist ein error code.

`test` ist wie eine if-Bedingung, wobei das Ergenis in den Exit Status geschrieben wird. Man kann auch `[...]` verwenden.

`exit` setzt den Status im Skript. `&&` sollte man nur bei einem Erfolg ausführen und `||` nur bei Fehlern ausführen.

## 1.Übung (Datei zu groß)

**Schreibe ein Skript mit einer Datei als Argument, es soll der Text `Datei ist zu groß` ausgegeben werden wenn die Datei mehr als 100 Bytes hat. Die Dateigröße kann mit `ls -l` und anschließendes cut ermittelt werden.**

**V1.1: Wenn die Datei nicht mehr als 100 Bytes hat soll der Text Größe OK ausgegeben werden.**

---

Also zuerst müssen wir ein Skript schreiben:
```
#!/bin/bash
 
size=$(ls -l "$1" | cut -d " " -f5)
 
test "$size" -gt 100 && echo "Datei ist zu groß" || echo "Größe OK"
```

Zuerst wird hier die Größe der Datei geholt und dann wird das 5. Feld mit `cut` geholt. Alles wird in die Variable `size` geschrieben. 

Danach wird mit `test` geprüft ob die Datei größer als 100 ist und wenn ja dann ist der exit status `0` und wenn nein dann ist der exit status `nicht gleich 0`.

Dann mit `chmod +x dateizugross.sh` ausführbar machen und dann mit `./dateizugross.sh test.txt` ausführen. Dann kommt das gewünschte Ergebnis `Größe OK`

## 2.Übung (Dir Creator)

**Schreibe ein Skript das den Namen eines Verzeichnisses übergeben bekommt. Das Verzeichnis soll angelegt werden wenn dieses noch nicht existiert. Wenn es das Verzeichnis schon gibt soll eine Warnung und der Inhalt des Verzeichnis angezeigt werden.**

---

Es wird wieder eine Skript Datei erstellt mit dem Inhalt:
```
#!/bin/bash
 
test -d "$1" && (echo "Warnung: Verzeichnis existiert bereits"; ls "$1") || mkdir "$1"
```

Hier wird erstmal geprüft ob es das Verzeichnis überhaupt gibt wenn ja dann wechselt der exit Status auf `0` und wenn nein dann auf `nicht gleich 0`, also wird ein neuer Ordner erstellt.

Danach wieder ausführbar machen mit `chmod +x dircreator.sh` danach ausführen mit `./dircreator.sh testordner` und es wird ein neuer Ordner erstellt!

## 3.Übung (Stundenplan)

**Schreibe ein Bash Script das für den aktuellen Wochentag eine Kurzform des Stundenplans ausgibt:**
```
$ ./splan.sh
Es ist Donnerstag
  AM SYTB SYTB frei MEDT MEDT BESP BESP
  ```

**Hinweis: Mit `date` den aktuellen Wochentag ermitteln.**

---

Wieder ein Skript erstellen mit dem Inhalt:
```
#!/bin/bash

tag=$(date +%A)

echo "Es ist $tag"

test "$tag" = "Monday" && echo " AM SYTB SYTB frei MEDT MEDT BESP BESP"
test "$tag" = "Tuesday" && echo " AM SYTB SYTB frei MEDT MEDT BESP BESP"
test "$tag" = "Wednesday" && echo " AM SYTB SYTB frei MEDT MEDT BESP BESP"
test "$tag" = "Thursday" && echo " AM SYTB SYTB frei MEDT MEDT BESP BESP"
test "$tag" = "Friday" && echo " AM SYTB SYTB frei MEDT MEDT BESP BESP"
```

Hier wird erstmal der aktuelle Wochentag geholt und daraufhin wird der Tag geprüft und mit `&&` wird der Stundenplan ausgegeben aber nur wenn es passt.

Dann wieder `chmod +x stundenplan.sh` und `./stundenplan.sh` und da kommt dann raus:
```
┌──(kali㉿kali)-[~]
└─$ ./stundenplan.sh 
Es ist Tuesday
 AM SYTB SYTB frei MEDT MEDT BESP BESP
```

## 4.Übung (HTML Generator)

**Erstelle ein Skript das ein Markdown Dokument nach HTML konvertiert:**
```
$ ./md2html test.md test.html
```
**Verwende `pandoc` zum konvertieren.**

**Dabei soll aber nur dann konvertiert werden wenn dies wirklich notwendig ist, d.h. die html Zieldatei entweder nicht existiert oder älter ist als die Quelldatei, denn wenn sich die Quelldatei nicht geändert hat ist das konvertieren auch nicht notwendig.**

---

Wieder ein Skript erstellen:
```
#!/bin/bash
 
test ! -e "$2" && pandoc "$1" -o "$2"
test "$1" -nt "$2" && pandoc "$1" -o "$2"
```

Es wird erstmal geprüft ob die HTML Datei schon existiert und wenn ja wird es mit `pandoc` konvertiert. Danach wird geprüft, ob die Markdown Datei neuer als die HTML Datei ist, wenn ja dann wird wieder konvertiert.

Dann muss man erstmal `pandoc` installieren mit `sudo apt install pandoc`.

Danach die Markdown Dateie erstellen mit `nano test.md` und dann die Datei wieder ausführbar machen und dann auch ausführen:
```
┌──(kali㉿kali)-[~]
└─$ chmod +x htmlgenerator.sh

┌──(kali㉿kali)-[~]
└─$ ./htmlgenerator.sh test.md test.html
```

# 5.Übung (C-Programm)

**Schreibe und kompiliere ein C-Programm das 2 Argumente über die Kommandozeile erhält. Das Programm soll die Differenz ausgeben und im exit Status anzeigen ob die erste Zahl größer als die zweite Zahl war. Teste den exit Status mit `&&`. Lass dir dabei von genAI helfen.**

---

Man erstellt die Datei `diff.c` und schreibt den code:
```
#include <stdio.h>
#include <stdlib.h>

int main(int argc, char *argv[])
{
    int a = atoi(argv[1]);
    int b = atoi(argv[2]);

    int diff = a - b;

    printf("Die Differenz ist: %d\n", diff);

    if(b < a)
    {
        return 0;
    }
    else
    {
        return 1;
    }
}
```
Also es wird erstmal erstmal ein String in ein Integer umgewandelt, da in einer Kommandozeile die Arguemente immer Strings sind, mit Hilfe der Funktion `atoi` und dann wird die differenz zwischen den Integern berechnet und ausgegeben. Daraufhin wird geprüft welche Zahl größer ist. Wenn also die erste zahl größer ist, so liefert die Funktion 0, also`true`,zurück, wenn aber die zweitel Zahl größer ist liefert die Funktion 1, also `false` zurück.

Danach kompileirt man mit `gcc diff.c -o diff`. Danach führt man das Programm aus und das Ergebnis ist dann:
```
┌──(kali㉿kali)-[~]
└─$ ./diff 30 16                        
Die Differenz ist: 14
```

Um das `&&` zu testen, gibt man folgendes ein:
```
┌──(kali㉿kali)-[~]
└─$ ./diff 30 16 && echo "Erste Zahl = größer"
Die Differenz ist: 14
Erste Zahl = größer
```
In diesem Fall stimmt die Aussage ja, weil 30 ja größer ist als 16. Wenn dies allerdings nicht der Fall ist, wird das echo gar nicht ausgegeben, wie hier:
```
┌──(kali㉿kali)-[~]
└─$ ./diff 3 16 && echo "Erste Zahl = größer" 
Die Differenz ist: -13
```
Also funktioniert `&&`!
