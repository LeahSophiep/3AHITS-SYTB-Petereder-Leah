# Zusammenfassung der Theorie

Mit if-Anweisungen kann man in Bash Bedingungen prüfen. Je nachdem, ob die Bedingung erfüllt ist oder nicht, werden unterschiedliche Befehle ausgeführt.
```
if [ $1 -gt 100 ]
then
  echo "Hey that's a large number"
  pwd
fi
```

In diesem Beispiel wird geprüft, ob der erste übergebene Parameter größer als 100 ist. Wenn das stimmt, wird eine Meldung ausgegeben.

Die eckigen Klammern [ ] stehen für das test-Kommando.
```
if [ -n "$3" ]
then
  echo "Parameter #3 ist vorhanden"
else
  echo "Parameter #3 fehlt"
fi
```

Hier wird überprüft, ob ein dritter Parameter übergeben wurde.
-n bedeutet, dass die Variable nicht leer ist.
```
if [ -r $1 ] && [ -s $1 ]
```
Die Bedingung ist nur wahr, wenn beide Teile erfüllt sind.
```
if [ $USER = "bob" ] || [ $USER = "andy" ]
```
Hier reicht es, wenn eine der Bedingungen stimmt.
```
x=42
y=10
if (( x > y ))
then
  echo "x > y"
fi
```
Diese Schreibweise wird für Zahlen verwendet und ist übersichtlicher. Die Variablen brauchen hier kein $.

## 1. Übung (Maximum)

**Schreibe ein bash Skript das 2 Zahlen als Argumente aus der Kommandozeile übernimmt. Die größere der beiden Zahlen soll ausgegeben werden.**

---

Man erstellt wieder eine Skript Datei  und fügt diesen Inhalt hinzu:
```
#!/bin/bash

if (( $1 > $2 ))
then
  echo "$1"
else
  echo "$2"
fi
```

Hier wird geprüft welche Zahl größer ist und dann ausgegeben.

Dann macht man es wider ausführbar mit `chmod +x maximum.sh` und führt es dann mit zwei Zahlen aus: `./maximum.sh 13 20`.

Hierbei wird dann die größere von den beiden Zahlen ausgegeben:
```
┌──(kali㉿kali)-[~]
└─$ ./maximum.sh 13 20                       
20
```

## 2. Übung (Un-/Gerade)

**Schreibe ein Script das von einer als Argument übergebenen Zahl prüft ob sie gerade oder ungerade ist.**

---

Man erstellt wieder eine Skript Datei und schreibt in den Inhalt folgendes:
```
#!/bin/bash

if (( $1 % 2 == 0 ))
then
  echo "Diese Zahl ist gerade"
else
  echo "Diese Zahl ist ungerade"
fi
```

Hier wird geschaut, ob die Zahlt geteilt durch 2 einen Rest von 0 hat, denn dann ist sie gerade.

Diese Datei macht man dann wieder mit `chmod +x geradenichtgerade.sh` ausführbar und dann führt man sie wieder mit einer Zahl aus und schaut, ob diese gerade oder ungerade ist:
```
┌──(kali㉿kali)-[~]
└─$ ./geradenichtgerade.sh 9576
Diese Zahl ist gerade
```

## 3. Übung (Directory)

**Schreibe ein Skript `makedir.sh` das mit einem Argument aufgerufen wird**
```
$ ./makedir.sh xyz
```
**Ein Directory mit dem Namen `xyz` soll angelegt werden falls es nicht existiert. Im Directory lege eine Datei mit dem Namen `xyz.txt` und Inhalt `xyz` an.**

**Existiert das Directory bereits so soll gefragt werden ob das Directory gelöscht werden darf. Bei dieser Abfrage soll angegeben werden wie viele Files sich im Directory befinden.**
```
Soll das Directory "xyz" (mit 5 Files) gelöscht werden? [j|n]:
```

- **Auswahl j: Das Directory wird gelöscht und wieder wie oben angelegt.**
- **Auswahl n: Das Skript wird beendet.**

---

Man erstellt wieder eine Skript Datei mit dem Namen `makedir.sh` und fügt diesen Inhalt rein:
```
#!/bin/bash

dir=$1

if [ -d "$dir" ]
then
  count=$(ls "$dir" | wc -l)

  echo -n "Soll das Directory \"$dir\" (mit $count Files) gelöscht werden? [j|n]: "
  read answer

  if [ "$answer" = "j" ]
  then
    rm -r "$dir"
    mkdir "$dir"
    echo "$dir" > "$dir/$dir.txt"
  else
    exit 0
  fi
else
  mkdir "$dir"
  echo "$dir" > "$dir/$dir.txt"
fi
```

Hier wird erstmal geschaut ob das Directory existiert, wenn ja dann soll die Anzahl der Dateien in diesem Directory gezählt werden und dann die Löschungsfrage gestellt werden.

Danach wieder geschaut was die Eingabe war, wenn es `j` war, so wird das Directory gelöscht. Wenn `n` gedrückt wurde, wird abgebrochen.

Wenn Nein dann soll ein neues Directory angelegt werden.

Man führt die Datei wieder mit `chmod +x makedir.sh` aus und dann führt man es mit dem Inhalt `xyz` aus.
```
──(kali㉿kali)-[~]
└─$ ./makedir.sh xyz           
                                                                             
┌──(kali㉿kali)-[~]
└─$ ./makedir.sh xyz
Soll das Directory "xyz" (mit 1 Files) gelöscht werden? [j|n]: j
                                                                             
┌──(kali㉿kali)-[~]
└─$ ./makedir.sh xyz
Soll das Directory "xyz" (mit 1 Files) gelöscht werden? [j|n]: n
```
Beim ersten Mal wird das Directory erstmal erstellt, deswegen kommt noch keine Frage. Wenn man es allerdings ein zweites Mal eingibt, kommt die Frage, da gesehen wurde dass das Directory existiert. Wenn man `j` klickt, wird das Directory gelöscht und bei `n`, bleibt alles gleich!

## 4. Übung (number lines script)

**Schreibe ein Skript `number.sh` das von der Kommandozeile aus aufgerufen werden kann.**

**Angenommen es gibt die Datei `test.txt` mit Inhalt:**
```
bbb eins
ccc zwei
aaa drei
```
**Grundsätzlich kann das Skript mit einer Datei als Argument ausgeführt werden:**
```
$ ./number test.txt
$ cat test.txt
```
```
1 bbb eins
2 ccc zwei
3 aaa drei
```

**Es werden die einzelnen Zeilen nummeriert (dafür kann das Tool `nl` verwendet werden). Achtung: das Skript soll die Datei verändern nicht ausgeben.**

**Sollte die Datei nicht exisitieren, nicht lesbar sein oder leer sein soll ein entsprechender Fehlertext ausgegeben werden.**

**Es soll weiters auch möglich sein dem Tool Daten über `stdin` zu übergeben (wenn keine Datei angegeben wurde):**
```
$ sort test.txt | ./number.sh
```
```
1 aaa drei
2 bbb eins
3 ccc zwei
```
**Hinweis: `/dev/stdin` kann in der Kommandozeile wie ein Dateiname verwendet werden und liefert die Daten von stdin.**

**Die Verwendung der Option `-?` soll dann noch zur Ausgabe eines Hilfetexts führen:**
```
number.sh: number lines of input
usage: number.sh [FILE]
FILE ... path to readable file of nonzero length
if FILE is omitted data is read from standard input
```
**Hinweis: der Befehl `exit` beendet sofort die Skriptabarbeitung.**

---

Eine Skript-Datei `number.sh` erstellen, mit dem Inhalt:
```
#!/bin/bash

if [ "$1" = "-?" ]
then
  echo "number.sh: number lines of input"
  echo "usage: number.sh [FILE]"
  echo "FILE ... path to readable file of nonzero length"
  echo "if FILE is omitted data is read from standard input"
  exit 0
fi

if [ -z "$1" ]
then
  nl /dev/stdin
  exit 0
fi

file=$1

if [ ! -f "$file" ]
then
  echo "Datei existiert nicht"
  exit 1
fi

if [ ! -r "$file" ]
then
  echo "Datei nicht lesbar"
  exit 1
fi

if [ ! -s "$file" ]
then
  echo "Datei ist leer"
  exit 1
fi

tmp=$(mktemp)
nl "$file" > "$tmp"
mv "$tmp" "$file"
```

Wenn man `-?` eingibt, bekommt man die Hilfestellung. Wenn kein Argument beigefügt ist, wird `stdin` verwendet. Dann werden die jeweiligen Fehler geprüft und dann werden die Dateien numeriert.

Danach erstellt man die Datei `test.txt` mit dem Inhalt:
```
bbb eins
ccc zwei
aaa drei
```

Man macht die Datei erstmal wieder ausführbar mit `chmod +x number.sh`, danach wird sie mit `test.txt` ausgeführt:
```
┌──(kali㉿kali)-[~]
└─$ ./number.sh test.txt

┌──(kali㉿kali)-[~]
└─$ cat test.txt      
     1  bbb eins
     2  ccc zwei
     3  aaa drei 
```
Also der Inhalt wurde numeriert!

Und auch wenn man `-?` eingibt, kommt die Hilfestellung:
```
┌──(kali㉿kali)-[~]
└─$ ./number.sh -?      
number.sh: number lines of input
usage: number.sh [FILE]
FILE ... path to readable file of nonzero length
if FILE is omitted data is read from standard input
```

## 5. Übung (Stundenplan V2)

**Schreibe ein Skript das aufgrund der aktuellen Zeit das aktuell Fach laut Stundenplan ausgibt. Pausen sind vereinfacht einem Fach zuzurechnen. Berücksichtige nur einen Wochentag, d.h. das Skript funktioniert nur an einem Donnerstag.**
```
$ ./splan2.sh
Donnerstag 11:00 SYTB
```
---
Man erstellt eine Skript-Datei `thursday.sh` mit dem Inhalt:
```
#!/bin/bash

tag=$(date +%A)
zeit=$(date +%H%M)

if [ "$tag" != "Thursday" ] && [ "$tag" != "Donnerstag" ]
then
  echo "Dieses Skript funktioniert nur am Donnerstag."
  exit 1
fi

if (( zeit >= 750 && zeit < 840 ))
then
  fach="Physik"
elif (( zeit >= 840 && zeit < 935 ))
then
  fach="AM"
elif (( zeit >= 935 && zeit < 1030 ))
then
  fach="SYTE"
elif (( zeit >= 1030 && zeit < 1135 ))
then
  fach="ITP2PM"
elif (( zeit >= 1135 && zeit < 1230 ))
then
  fach="Turnen"
elif (( zeit >= 1230 && zeit < 1325 ))
then
  fach="Turnen"
elif (( zeit >= 1325 && zeit < 1510 ))
then
  fach="Deutsch"
else
  fach="kein Unterricht"
fi

echo "Donnerstag $(date +%H:%M) $fach"
```

Zuerst einmal wird der Tag sowie die Uhrzeit auf eine Variable gespeicht, danach wird geschaut, ob heute überhaupt Donnerstag ist, um dieses Skript auszuführen, wenn nicht Donnerstag ist, so wird abgebrochen. Danach wird geschaut welche Uhrzeit es gerade hat und je nachdem wird das Fach ausgegeben.

Danach macht man es mit `chmod +x thursday.sh` ausführbar und führt dann das Skript aus:
```
┌──(kali㉿kali)-[~]
└─$ ./thursday.sh 
Dieses Skript funktioniert nur am Donnerstag.
```
Da heute nicht Donnerstag ist, wird diese Meldung ausgegeben und abgebrochen.

Wenn ich den Tag allerdings auf `Tuesday` und `Dienstag` stelle, bekommt man diese Ausgabe:
```
┌──(kali㉿kali)-[~]
└─$ ./thursday.sh
Dienstag 09:21 AM
```
