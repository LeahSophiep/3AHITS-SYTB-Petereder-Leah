# 1. Zusammenfassung der Theorie

Es wurden die Linux-Befehle `grep` und `sed` verwendet. Diese Ausdrücke ermöglichen es, flexibel nach bestimmten Mustern in Dateien zu suchen oder Texte zu verändern. 

Mit `grep` können bestimmte Inhalte aus Dateien herausgefiltert werden, wie hier:

```
grep 'Werkstatt' klassenkassa.csv
```

zeigt alle Zeilen an, die das Wort „Werkstatt“ enthalten.

Der Punkt `.` steht in Regex für ein beliebiges Zeichen:

```
grep 'W.' klassenkassa.csv
```
Hier werden daher alle Wörter gefunden, die mit `W` beginnen und mindestens ein weiteres Zeichen besitzen, z. B. „Walter“ oder „Werkstatt“:
```
Sophia,22.09.2023,15.0,Werkstatt
Lena,18.09.2023,13.0,Werkstatt
Emma,22.09.2023,42.0,Werkstatt
Nico,18.09.2023,23.0,Werkstatt
Timo,25.11.2023,5.0,Walter
```

Das Zeichen `*` bedeutet, dass das vorherige Zeichen beliebig oft vorkommen darf, also auch 0-mal:

```
grep 'W.*' klassenkassa.csv
```

findet alle Wörter, die mit `W` beginnen:
```
Laura,28.10.2024,3.21,Werkstatt
Jan,30.11.2024,2.56,Walter
Benjamin,25.11.2024,15.78,Werkstatt
Paul,14.11.2024,11.92,Walter
```

Das Zeichen `+` bedeutet, dass das Zeichen davor mindestens einmal vorkommen muss. `sed` versteht `+` jedoch nur mit `-E` für expandend:
```
┌──(kali㉿kali)-[~]
└─$ echo 'jhabbbaabgfah' | sed -E 's/[ab]+/XYZ/'
jhXYZgfah
```

Mit `[ ]` werden Zeichenklassen definiert:
```
┌──(kali㉿kali)-[~]
└─$ grep '1[0-9]\.[0-9][0-9]\.' klassenkassa.csv
Lena,18.09.2023,13.0,Werkstatt
Lukas,15.09.2023,2.5,Kino
Nico,18.09.2023,23.0,Werkstat
```
sucht erstmal nach Zahlen von 10-19 mit Punkt dahinter und daraufhin zwei Zahlen von 0-9.

Mit `grep -o` wird nur der eigentliche Treffer ausgegeben und nicht die gesamte Zeile:
```
┌──(kali㉿kali)-[~]
└─$ grep -o '1[0-9]\.' klassenkassa.csv
15.
18.
13.
15.
18.
10.
```

Außerdem wird `sed` verwendet, um Texte zu ersetzen.:
```
┌──(kali㉿kali)-[~]
└─$ echo '1234ccbbaa1234' | sed 's/ccbbaa/XYZ/'
1234XYZ1234
```

Der Befehl `sed` funktioniert folgendermaßen:

```
s/ALT/NEU/
```

wobei `s` für „substitute“ (= ersetzen) steht.

Mit Zeichenklassen wie `[ab]` kann entweder nach `a` oder `b` gesucht werden.
Das Zeichen `^` steht für den Anfang der Zeile und `$` steht für das Ende der Zeile:
```
┌──(kali㉿kali)-[~]
└─$ echo 'jhdfkjh' | sed 's/^j/_/' 
_hdfkjh
```

```
┌──(kali㉿kali)-[~]
└─$ echo 'jhdfkj' | sed 's/j$/_/' 
jhdfk_
```
Dadurch können Suchmuster genauer eingeschränkt werden.

 ## 3. Übung (Subdir Count)

 Schreibe einen shell Einzeiler mit dem man die Anzahl der Unterverzeichnisse im aktuellen Verzeichnisse zählt.

Tipp: Directories haben in der `ls -l` Ausgabe ganz am Beginn ein `d`.

---

Die Lösung ist:
```
┌──(kali㉿kali)-[~]
└─$ ls -l | grep '^d' | wc -l
15
```

Man kommt darauf, da jedes Verzeichnis mit einem `d` anfängt, filert man mit `^d`, da `^` ja das Zeichen für den Anfang einer Zeile ist.

mit `wc -l` zählt man die Anzahl der Zeilen und `ls -l` zeigt alle Dateien und Verzeichnisse an.
